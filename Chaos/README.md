# Хаос 💥

## О чём?

Рассматривается логистическое уравнение x_n+1 = r * x_n * (1 - x_n).

Было доказано, что при r = 4 динамика логистического уравнения обладает хаосом.
Это, в частности, означает чувствительность к начальным данным. Значит, если мы
возьмём две одинаковые точки x_0 = y_0 = 2/7 (например), то после некоторого
числа итераций мы получим разные результаты, которые максимально могут отличаться
на 1.

## Как?

Для этого также используется тот факт, что в компьютере операции с плавающей точкой
умножения считается не абсолютно точно, а какая-то часть отбрасывается. В примере
я беру точку 2/7 и одну итерирую по формуле x = 4 * x - 4 * x * x, а другую x = 4 * x * (1 - x). То что откидывается
потом сыграет свою роль и результаты разойдутся,
так как любая, даже очень малая ошибка, приводит к существенно неправильному ответу.

Добавлю, что любой и без хаоса эти результаты не будут абсолютно точными, но будут расходиться
намного медленнее.

Для примера используется уравнение с r = 3, для того чтобы показать также вариант,
в котором хаоса нет, и значит ошибка компьютера будет незначительной, даже после
большого количества итераций.

Результаты можно увидеть, если запустить программу.

## Запуск 🚀

Запустить программу можно двумя способами: в первом случае будет использован maven, а во втором - нет.

```shell
# Собираем .jar-файл;
mvn clean package;

# Запускаем .jar-файл на исполнение;
java -jar target\Chaos.jar;
```

или же

```shell
# Компилируем в байт-код Main-класс;
javac -d out src/main/java/a/slelin/work/math/chaos/Main.java;

# Запускаем класс;
java -cp out a.slelin.work.math.chaos.Main;

# Не забываем почистить в конце .class-файлы;
rm -r out;
```
